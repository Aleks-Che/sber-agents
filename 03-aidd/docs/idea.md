# Идея: LLM-ассистент "Кулинарный помощник" в Telegram

## Общее описание

Разработать Telegram-бота, который использует крупную языковую модель (LLM) для ведения диалога с пользователем в роли **Кулинарного помощника**. Бот должен понимать кулинарные запросы, давать рецепты, советы по приготовлению, помогать с подбором ингредиентов, отвечать на вопросы о питании и т.д.

## Цель

Создать удобного, интерактивного ассистента, который:

- Отвечает на вопросы пользователя в естественной форме.
- Предоставляет персонализированные кулинарные рекомендации.
- Помогает в планировании меню, расчёте порций, конвертации единиц измерения.
- Может вести контекстный диалог, запоминая предыдущие сообщения.

## Архитектура

1. **Telegram Bot API** – интерфейс для общения с пользователем.
2. **LLM Backend** – сервис, который обрабатывает промпты и генерирует ответы. Можно использовать:
   - OpenAI GPT (через API)
   - локальную модель (например, Llama, Mistral) через Ollama или аналоги
   - YandexGPT или другие облачные LLM
3. **Системный промпт** – задаёт роль и стиль общения бота.
4. **Контекстное хранилище** – временное хранение истории диалога для каждого пользователя.
5. **Дополнительные модули** (опционально):
   - База рецептов (интеграция с внешними API, например, Spoonacular).
   - Управление диалогом (FSM для сложных сценариев).

## Системный промпт (пример)

```
Ты — Кулинарный помощник, эксперт в области кулинарии, питания и гастрономии. Твоя задача — помогать пользователям с любыми вопросами, связанными с приготовлением пищи.

Ты должен:
- Давать чёткие, практичные советы и рецепты.
- Учитывать диетические ограничения (вегетарианство, аллергии, диеты).
- Предлагать замены ингредиентов, если чего-то нет под рукой.
- Объяснять шаги приготовления простым языком.
- Поддерживать дружелюбный, мотивирующий тон.
- Избегать вредных или опасных рекомендаций.
- Если вопрос не по кулинарии, вежливо направлять разговор в соответствующее русло.

Формат ответов: текст с эмодзи для наглядности, списки, где уместно.
```

## Основные функции

- **Диалог с LLM**: Пользователь пишет сообщение, бот формирует промпт с историей и отправляет в LLM, возвращает ответ.
- **Команды**:
  - `/start` – приветствие и краткое описание возможностей.
  - `/help` – список команд и примеры запросов.
  - `/recipe` – быстрый поиск рецепта по названию или ингредиентам.
  - `/reset` – очистка контекста диалога.
- **Контекст**: Хранение последних N сообщений для каждого чата (например, в Redis или in‑memory).
- **Мультиязычность**: Поддержка русского и английского (опционально).

## Технологический стек (предварительно)

- **Язык**: Python (библиотеки `python-telegram-bot`, `aiogram` или `telebot`).
- **LLM интеграция**: `openai`, `ollama`, или прямой HTTP‑запрос к API.
- **Хранение контекста**: `redis` для продакшена, `dict` в памяти для разработки.
- **Деплой**: Docker, облачная платформа (Heroku, VPS, Yandex Cloud).

## План разработки

1. Прототип: простой бот, отвечающий через OpenAI API.
2. Добавление контекста диалога.
3. Расширение команд и улучшение UX.
4. Интеграция с внешними API (рецепты, картинки).
5. Тестирование, оптимизация, деплой.

## Риски и ограничения

- Задержки ответа из‑за LLM API.
- Токены/стоимость запросов.
- Непредсказуемость LLM (нужна пост‑обработка ответов).
- Конфиденциальность данных (не хранить персональную информацию).

## Дальнейшее развитие

- Голосовой ввод/вывод.
- Генерация изображений блюд.
- Планировщик питания на неделю.
